# 🚀 多语言国际化快速启动指南

## 📋 立即开始的 3 个步骤

### Step 1: 创建基础架构（30分钟）

#### 1.1 创建目录结构
```bash
cd frontend
mkdir -p i18n/locales
mkdir -p common
```

#### 1.2 创建核心文件
- `i18n/index.js` - i18n 核心引擎
- `i18n/locales/zh.json` - 中文语言包
- `i18n/locales/en.json` - 英文语言包
- `common/i18n-helper.js` - 辅助函数

---

### Step 2: 实现示例页面（1小时）

#### 2.1 改造商城首页
将 `/mall/index.html` 改造为支持多语言

**改造前**:
```html
<h1>非遗商城</h1>
```

**改造后**:
```html
<h1 data-i18n="mall.title">非遗商城</h1>

<script src="/i18n/index.js"></script>
<script>
  window.addEventListener('DOMContentLoaded', async () => {
    await i18n.init(i18n.autoDetectLocale());
    translatePage();
  });
</script>
```

---

### Step 3: 测试和验证（30分钟）

#### 3.1 测试语言切换
```javascript
// 在浏览器控制台测试
i18n.setLocale('en'); // 切换到英文
i18n.setLocale('zh'); // 切换回中文
```

#### 3.2 验证翻译
- 检查所有文本是否正确翻译
- 测试语言切换是否流畅
- 验证本地存储是否保存语言偏好

## 🛠️ 技术选型

### 翻译服务对比

| 服务 | 优点 | 缺点 | 成本 | 推荐度 |
|------|------|------|------|--------|
| **ChatGPT/Claude** | 质量高、理解上下文 | 需要人工校对 | ¥0.01/字 | ⭐⭐⭐⭐⭐ |
| **Google Translate API** | 快速、便宜 | 质量一般 | ¥0.005/字 | ⭐⭐⭐ |
| **DeepL** | 质量高 | 语言支持少 | ¥0.02/字 | ⭐⭐⭐⭐ |
| **人工翻译** | 质量最高 | 贵、慢 | ¥0.15-0.20/字 | ⭐⭐⭐⭐ |
| **AI + 人工校对** | 性价比高 | 需要流程管理 | ¥0.03/字 | ⭐⭐⭐⭐⭐ |

**推荐**: ChatGPT/Claude + 人工校对

## 💰 成本优化建议

### 1. 使用 AI 翻译
- **ChatGPT API**: $0.002/1K tokens ≈ ¥0.01/字
- **Claude API**: $0.003/1K tokens ≈ ¥0.015/字

### 2. 批量翻译
```javascript
// 一次性翻译整个语言包
const prompt = `
请将以下 JSON 翻译成英文，保持 JSON 结构不变：

${JSON.stringify(zhLanguagePack, null, 2)}

要求：
1. 保持 key 不变
2. 只翻译 value
3. 保持 JSON 格式
4. 确保翻译准确、流畅
`;
```

### 3. 人工校对重点
- 品牌名称
- 专业术语
- 文化相关内容
- 关键转化点（购买按钮等）

**预估**: 人工校对 10% 内容 = ¥5,000

## 🎨 UI/UX 优化

### 语言切换器设计

#### 位置选项
1. **顶部导航栏**（推荐）
   - 显眼、易于访问
   - 不影响主要内容

2. **底部导航栏**
   - 移动端友好
   - 节省顶部空间

3. **浮动按钮**
   - 始终可见
   - 可能遮挡内容

#### 样式示例
```html
<div class="language-switcher">
  <button class="lang-btn">
    🌍 中文 ▼
  </button>
  <div class="lang-dropdown">
    <button data-locale="zh">🇨🇳 中文</button>
    <button data-locale="en">🇺🇸 English</button>
    <button data-locale="ja">🇯🇵 日本語</button>
    <button data-locale="fr">🇫🇷 Français</button>

    <button data-locale="ru">🇷🇺 Русский</button>
```

## 🚨 风险和应对

### 风险 1: 翻译质量不佳
**应对**: 
- AI 翻译 + 人工校对
- 建立翻译质量标准
- 用户反馈机制

### 风险 2: 开发周期延长
**应对**:
- 采用渐进式实施
- 优先核心页面
- 并行开发和翻译

### 风险 3: 维护成本高
**应对**:
- 建立标准化流程
- 使用翻译管理工具
- 定期批量更新

### 风险 4: 性能影响
**应对**:
- 按需加载语言包
- CDN 加速
- 缓存策略
